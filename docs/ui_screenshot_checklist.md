# UI Screenshot Checklist

Use this with `scripts/ui_build_and_shoot.sh` and keep the target state explicit before each capture.

## Target

- Target screen:
- Route name: `suggestion`, `review`, `conflict`, `finalized`, `groceryList`, `settings`, or `familySettings`
- Expected device:
- App state seed: launch route only, plus `UITEST_RESET=1` when needed

## Acceptance Items

- [ ] The screenshot opens on the intended screen without manual navigation.
- [ ] The title and primary content match the chosen route.
- [ ] No blocking alerts, placeholders, or loading indicators are visible.
- [ ] `artifacts/ui-latest.png` and `artifacts/ui-latest.json` were regenerated by the latest run.

## Route Notes For This Repo

- `suggestion` maps to the suggestion screen.
- `review` maps to the review screen.
- `conflict` maps to the conflict screen.
- `finalized` maps to the finalized weekly summary.
- `groceryList` maps to the grocery list screen.
- `settings` maps to the settings screen.
- `familySettings` maps to the family settings screen.
- The script passes `-ui_debug_route <route>` directly, so the visual loop stays aligned with the app's debug route contract.
